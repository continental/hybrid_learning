ActivationDatasetWrapper¶

class hybrid_learning.datasets.activations_handle.ActivationDatasetWrapper(dataset, layer_key=None, model_description=None, act_map_gen=None, act_map_filepath_fn=None, force_rebuild=False, activations_root=None, lazy_generation=True, **data_args)[source]¶

Bases: hybrid_learning.datasets.base.BaseDataset

Wrapper for image datasets that will generate and yield activation maps. Behaves like a sequence of tuples of:

Activation map

Activation map tensor of the given model and the given layer for image at the corresponding index in the wrapped dataset

Ground Truth

Ground truth at the corresponding index in the wrapped dataset

If the activation map generator is given during init with lazy_generation set to True, the data is lazily generated, i.e. any call to __getitem__() (resp. getitem()) will generate the corresponding activation map if it does not yet exist. To generate all activation maps, call generate_act_maps(), but be aware that this can be very time consuming depending on the generator.

static _default_act_map_filepath_fn(i, dataset)[source]¶

Obtain the act map filepath relative to the activations root. As activations root, act_maps_root is used. It is assumed that

  • the images are saved as unique_id.{png,jpg}, that

  • either dataset or dataset.dataset (e.g. in case of a torch.utils.data.Subset instance) provide a function image_filepath(i: int), and that

  • this image_filepath function yields some/path/to/img/unique_id.{png,jpg} for the image at position i, with unique_id being stable against shuffling of the indices i.

Then the activation map filepath is given as unique_id.pkl (path relative to an activation map root).

act_map_filepath(i)[source]¶

Return the path to the activation map file. The base directory is act_maps_root. The basename is:

  • If :py:attr:`dataset` or its member ``dataset`` features a ``image_filepath()`` method: the same basename as the original input image, with .png resp. .jpg replaced by .pkl

  • Else: the zero-padded decimal index number ending with .pkl

Warning

Make sure the used act_map_filepath_fn yields unique IDs for all elements of the dataset. Otherwise, the wrong activation maps may be loaded and duplicates overwritten.

Parameters

i (int) – index of the image to get activation map for.

property _default_activations_root¶

Default root folder for storing activation map files.

act_map_filepath_fn: Callable[[int, BaseDataset], str]¶

The function to determine the activation map filepath relative to the given activations root at index i. The activations root act_maps_root is used. It must accept the index and the original dataset.

Note

It should be stable against reindexing of the original dataset like shuffling or subsetting!

The default requires an image_filepath function in dataset or dataset.dataset that yields unique image file names. See _default_act_map_filepath_fn().

act_map_gen: torch.nn.Module¶

Callable that yields an activation map given a valid input datum. Input data is assumed to origin from the original dataset. Used to generate missing activation maps in getitem(). If lazy_generation=False, all missing activation maps are generated on __init__ call. If additionally force_rebuild=True, also existing ones are replaced.

act_maps_root: str¶

Root directory under which to store and locate activation maps.

dataset: BaseDataset¶

The dataset to wrap; activation maps are created from input; the ground truth is equal to the original one.

layer_key: Optional[str]¶

Optional description of the layer from which the activation maps were retrieved.

model_description: str¶

Description of the model with which activation maps were generated. Used to create default activation map root name (by creating form of hash).
