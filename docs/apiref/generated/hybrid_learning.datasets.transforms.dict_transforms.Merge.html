Merge¶

class hybrid_learning.datasets.transforms.dict_transforms.Merge(*in_keys, out_key=None, overwrite=True, skip_none=True, replace_none=None)[source]¶

Bases: hybrid_learning.datasets.transforms.dict_transforms.DictTransform, abc.ABC

Merge the binary masks / scalars of the input keys and store them under the specified output key. The merge operation may recursively have other child merge operation as in_keys, which are evaluated on the given dictionary before the parent is.

Operation

The actual operation is hidden in the __call__() method: It is given a dictionary of annotations of the form {ID: value} and will return the dict with the merged mask added as {out_key: value}. The intermediate outputs of all child operations are also added, so make sure to remove them afterwards if they are not needed anymore. The benefit of this is that results may be reused amongst different operations.

Format

The (recursive) merge operation must be specified in conjunctive normal form, i.e. of the form

AND(..., [NOT(...), ...], [OR(..., [NOT(..), ...])])

(see https://en.wikipedia.org/wiki/Conjunctive_normal_form). Available operations are AND (intersection), OR (union), and NOT (inversion). Boolean classification labels are treated as all-one-masks. One can use Merge.parse() to parse a string representation of an operation tree. There, again, the formula must be in conjunctive normal form and connectors must be encoded as follows:

  • AND: a&&b

  • OR: a||b

  • NOT: ~a

Examples

To get all heads, noses, and mouths of real persons in bathrooms, call:

>>> a = AND("person", OR("head", "nose", "mouth"), NOT("bathroom"))
>>> a == Merge.parse("person&&head||nose||mouth&&~bathroom")
True

__call__(annotations)[source]¶

Apply this operation to the annotations dict. The operation of this instance is defined in operation. First apply all child operations to the dict. Hereby try to overwrite a value of annotations if its key correspond to an out_key of a child operation, but do not create the value of a key twice. Then apply operation on the originally given and generated values now stored in annotations and store the result also in annotations.

__eq__(other)[source]¶

Two merge operations are considered equals, if their (normalized) representations coincide. (See normalized_repr()). This means, they recursively have the same children up to commutation.

Warning

Duplicate children are not filtered for now, i.e. AND("a") != AND("a", "a")

property operation_keys¶

The keys used for this parent operation (constants and children output keys). All consts and the out_key of all children operations.

out_key: str¶

The key to use to store the merge output in the annotations dict. Take care to not accidentally overwrite existing keys (cf. overwrite).

overwrite: bool¶

Whether to overwrite a value in the input dictionary when applying this operation. The operation is defined in operation(). The key that may be overwritten is stored in out_key. An exception is raised if this is False and the key exists.

replace_none: Optional[Any]¶

If not None, any received None value is replaced by the given value. Key-value pairs with None value may come from the input or from child operations.

skip_none: bool¶

If set to True, when a None input value is encountered simply None is returned. If False, an error is raised.
